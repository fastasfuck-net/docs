FastAsFuck Documentation

Ultimate reverse proxy and DDoS protection service for Minecraft servers. This documentation will guide you through setup, configuration, and advanced features.

Info:

FastAsFuck provides enterprise-grade DDoS protection with minimal latency impact (<10ms) and 99.9% uptime guarantee.

Key Features

  • Reverse Proxy Protection - Hide your real server IP
  • DDoS Mitigation - Block Layer 3/4/7 attacks automatically
  • Global Network - Multiple edge locations worldwide
  • Real-time Monitoring - 24/7 attack detection and blocking
  • Custom MOTD - Maintain player engagement during downtime
  • Analytics Dashboard - Detailed traffic and attack statistics
  • API Access - Programmatic control and monitoring

Prerequisites

Before setting up FastAsFuck protection, ensure you have:

Server Requirements

  • Minecraft server running on a dedicated server or VPS
  • Root/administrator access to your server
  • Static IP address for your server
  • Domain name with DNS management access

Supported Versions

  • Minecraft Java Edition (all versions)
  • Minecraft Bedrock Edition
  • Paper, Spigot, Bukkit, Forge, Fabric servers
  • Proxy servers (BungeeCord, Waterfall, Velocity)
Warning:

Shared hosting services are not supported. You need full control over your server configuration.

Quick Setup

Get your server protected in under 30 minutes with this step-by-step guide.

Step 1: Contact FastAsFuck Team

Reach out through our contact form with the following information:

  • Server IP address and port
  • Domain name you want to protect
  • Expected concurrent players
  • Current hosting provider

Step 2: Receive Protection Details

Within 24 hours, you'll receive:

  • Protected IP address
  • DNS configuration instructions
  • Dashboard access credentials
  • Configuration checklist

Step 3: Update DNS Records

Update your domain's DNS records to point to the protected IP:

# Before (example) play.yourserver.com A 203.0.113.10 # After play.yourserver.com A 185.xxx.xxx.xxx # Protected IP provided by FastAsFuck

Step 4: Verify Protection

Once DNS propagates (1-6 hours), verify protection is active:

  • Connect to your server using the domain
  • Check dashboard for incoming traffic
  • Test server responsiveness

DNS Setup

Detailed DNS configuration for various providers and scenarios.

Standard A Record Setup

For most Minecraft servers, you'll configure a simple A record:

# Main server record play.yourserver.com A 185.xxx.xxx.xxx # Optional subdomain records lobby.yourserver.com A 185.xxx.xxx.xxx creative.yourserver.com A 185.xxx.xxx.xxx

SRV Record Configuration

If using non-standard ports, configure SRV records:

# SRV record format: _service._protocol.name TTL class SRV priority weight port target _minecraft._tcp.play.yourserver.com SRV 0 5 25565 protected.fastasfuck.net

Provider-Specific Instructions

Cloudflare

  1. Log into Cloudflare Dashboard
  2. Select your domain
  3. Go to DNS settings
  4. Add/Edit A record with protected IP
  5. Important: Set proxy status to "DNS only" (grey cloud)
Critical:

Never enable Cloudflare proxy (orange cloud) for Minecraft records. It will break connections.

Namecheap

  1. Login to Namecheap account
  2. Go to Domain List
  3. Click "Manage" next to your domain
  4. Go to "Advanced DNS" tab
  5. Add/modify A record with protected IP

Server Configuration

Configure your Minecraft server to work optimally with FastAsFuck protection.

Server Properties

Update your server.properties file:

# Bind to all interfaces to accept proxied connections server-ip=0.0.0.0 server-port=25565 # Enable white-list for additional security white-list=true enforce-whitelist=true # Query settings for monitoring enable-query=true query.port=25565

IP Forwarding (BungeeCord/Velocity)

For proxy setups, enable IP forwarding to preserve real player IPs:

# BungeeCord config.yml ip_forward: true prevent_proxy_connections: false # Spigot/Paper spigot.yml settings: bungeecord: true

Firewall Configuration

Configure server firewall to only accept connections from FastAsFuck IPs:

# UFW (Ubuntu) sudo ufw deny 25565 sudo ufw allow from 185.xxx.xxx.0/24 to any port 25565 # iptables iptables -A INPUT -p tcp --dport 25565 -s 185.xxx.xxx.0/24 -j ACCEPT iptables -A INPUT -p tcp --dport 25565 -j DROP

Common Issues

Solutions to frequently encountered problems.

Connection Issues

Problem: Players can't connect to server

Possible causes:

  • DNS not fully propagated
  • Firewall blocking FastAsFuck IPs
  • Server not bound to correct interface

Solutions:

  1. Check DNS propagation: nslookup play.yourserver.com
  2. Verify firewall rules allow FastAsFuck IP ranges
  3. Ensure server-ip=0.0.0.0 in server.properties
  4. Test direct IP connection to verify server is running

Problem: High latency/lag

Diagnostic steps:

  1. Check dashboard for added latency metrics
  2. Test direct server connection vs. protected connection
  3. Review server resource usage (CPU, RAM, disk)
  4. Check for active DDoS attacks in dashboard

Dashboard Issues

Problem: Dashboard shows server offline

Check these items:

  • Server is actually running and accessible
  • Query is enabled in server.properties
  • No firewall blocking monitoring connections
  • Server responding to ping/query requests

Getting Help
